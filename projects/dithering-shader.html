Dithering Shader

C++ OpenGL

Overview

Final project of first computer graphics course, CSC 471 Intro to Computer Graphics, using OpenGL we were tasked with creating a scene with various interesting computer graphics effects. The effect I wanted to emulate was a dithering shading technique I saw implemented into a realtime 3D enviroment in the game Return of the Obra Dinn. With the guidence of a github project by user NBAyunjay, along with a wikipedia page covering Bayer matrices and a tutorial from writer Aeris on The Medium, I implemented a custom fragment shader that uses the effect for shading in my scene.

What is Dithering?

Ordered Dithering is a technique that achieves a smooth blend of values in an image by changing the frequancy of pixel colors through a patterned grid, achieved with only two colors to create depth in an image. My approach takes place in the fragement shader, which asks how dark a given area of geometry should be and assigns it a blending pattern between the two input colors to create a rich transition from one color (light) and the other (dark).

The dithering effect can be adjusted in the realtime enviroment with the 'o' and 'p' to show what different interpolation values do to the final image.

Scene Details

The scene was constructed of a 3D scan of St. Peters Basilica in Rome and a mesh of a hydra found for free use in an educational context on turbosquid. Basic 3D scene functions built up throughout the quarter are used, like a pitch and yaw camera, MVP matrices for models, directional scene lighting, Spline tours of the scene , and Camera restrictions in the form of a 'fly' mode and a 'walk' mode for scene traversal. Other tools built but not utilized in this final scene are a hierarchecal animation system, texture mapping, and point lights.
