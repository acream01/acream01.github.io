Vulkan Compute Shader Cloth Simulation

Overview

This project was a collaboration between me and Claire Ogawa for the final project of CSC 572 Graduate Computer Graphics. The intention was to build a realtime enviroment using the Vulkan Graphics Library, and to use a compute shader to create a simple cloth simulation. This was the first time either of us used vulkan, and so setting up the project from scratch was a massive, multi week long task.

Cloth simulation has traditionally been done using many different methods. One of the most efficient methods used in modern graphics programming is the utilization of GPU side compute shaders to calculate the physical interactions for a simulated cloth plane. Compute shaders can be implemented with every industry standard modern graphics library, and for this project we adapted the work of Va Hongly, Min-Hyung Choi, and Min Hong in Vulkan.

Cloth simulation is important for modeling garments in media, like films, video games, contributing to both realism and stylistic rendering. Modeling this behavior contributes to scene aesthetics, immersion, and believable physics interactions. It is implemented using “the mass-spring system,” and both implicit and adaptive constraints [6]. Our project will implement these physics mechanisms using compute shaders to construct a believable cloth simulation.

How it works

This project takes a plane and texture, applies according transforms, and runs through a shader pipeline to simulate cloth motion. The Shader takes data in the form of a UBO (Uniform Buffer Object) full of simulation parameters, then transforms the cloth based on its position, velocity, and spring constrains from the previous frame.

The use of the compute shader makes the simulation extremely performant on dedicated GPU hardware in real time. We included texture mapping and a key to flip the gravity and rotate the object to observe the results from different angles.
